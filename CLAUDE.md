# CTBC APAC Syndicated Loan Tracker — project context

Confidential internal tool for a CTBC Bank corporate-banking intern. A weekly dashboard + briefing on the Asia-Pacific syndicated loan market, with a Taiwan / CTBC lens. **See NOTICE.md — confidential, do not publish externally or push to a public remote.**

## Project layout (source → dist)
- **Edit `src/`, never `dist/`.** `src/template.html` holds all CSS + JS + page structure with a single `/*__DATA__*/` marker; `src/data/*.json` holds the data. The build injects the data at the marker to produce `dist/CTBC_Loan_Market_Dashboard.html` — the deliverable: ONE self-contained file (inline CSS + vanilla JS, no dependencies, opens by double-click, works offline).
- **Build & test:** `npm run build` (or `python3 tools/build.py` — a no-Node mirror) bundles template + data → dist. `npm test` runs the jsdom smoke test (`tools/test.mjs`: every week renders & switches, rows expand, sort/filter work, Trends + secondary render). `npm run verify` = build then test. First run: `npm install` (pulls jsdom). `tools/extract.py` was the one-off that split the original monolith — kept for provenance, not part of the loop.
- The dashboard has two views via the header **View** toggle: **Weekly** (per-reading snapshot with a date toggle, incl. a **world deal map**) and **Trends** (cross-week charts — volume, sector mix, theme tracker, CTBC involvement, pricing scatter).

## How the dashboard works (for edits)
- **Data lives in `src/data/`, injected at build.** One file per reading: `src/data/<week>.json` = `{key (e.g. "jun15"), tab (date-toggle label, e.g. "Jun 15"), date (ISO, for chronological sort), week:{…the week object…}}`. `src/data/shared.json` = `{wider, gloss, secondary}` (cross-week). `src/data/world_land.json` = the inlined map land rings. The build assembles `WEEKS` (keyed) and `ORDER` (sorted by `date`, newest last) from the week files — so `WEEKS`/`ORDER` exist at runtime exactly as before, just generated.
- Each `week` object has: `label`, `theme`, `lead`, `wnLabel`, `wnNote`, `whatsnew[]`, `kpis[5]`, `big[2]` (each `{title,cap,rows:[{lab,val,w,color}],note}`), `ctbcCap`, `ctbcReport[]`, `ladderCap`, `ladder[]`, `risks[]`, `deals[]`. A deal = `{borrower, country, sector, amtUSD (US$m or null), display, tenor, leads, pricing, ctbc (1 if CTBC is a lender/MLAB/owner else 0), theme, spreadBp (number or null), basis, lon, lat, loc (display "City, Country"), locType ("HQ"|"project"|"capital"|"target")}` — plus optional `flowFrom` (origin country, e.g. `"Taiwan"`) on CTBC cross-border deals, which draws a flow arc on the map. `lon`/`lat` place the deal's dot on the world map at the borrower's HQ (or the project/asset site for project finance; the entity's real location even if outside APAC).
  - `theme` = one of the keys in `THEME_META` (`ai_semis`, `ai_dc`, `brokerage`, `green_infra`, `property`, `ma_lbo`, `fin_leasing`, `corp_other`) — classify by the borrower's business / deal purpose (securities firm→brokerage; LBO/MBO/acquisition→ma_lbo; bank on-lending→fin_leasing even if SLL; infra assets→green_infra).
  - `spreadBp` = margin over the deal's own benchmark in bp (range→midpoint; below-LPR→negative; all-in if no over-benchmark figure; `null` if none). `basis` = benchmark name (`SOFR`/`Taibor`/`Hibor`/`all-in`/`1y LPR (below)`/… or `""`).
- `ORDER` (built chronologically from each file's `date`) drives the date toggle; `current = ORDER[ORDER.length-1].key` (always opens on the newest week).
- The **Trends** view computes all aggregates (volume, sector counts, theme volume/count, CTBC, scatter) live in JS from the per-deal `amtUSD`/`theme`/`spreadBp`/`ctbc` — no separate trend data to maintain. Theme/sector colours: `THEME_META` / `SECTOR_COL`.
- The **Deal map** (Weekly view, `renderMap()` from `renderWeek()`) is a pan/zoom **world** map: one clickable dot per deal at its `lon`/`lat`, sized by `amtUSD`, coloured by `SECTOR_COL`, ringed teal when CTBC-involved; teal arcs run Taiwan→borrower for deals tagged `flowFrom`. Land outlines are inlined as `WORLD_LAND` (Natural Earth **50m**, Douglas-Peucker-simplified, Pacific-centred) — no CDN. `projectW()` is a Pacific-centred equirectangular projection into a 1000×420 world space; `mapView` is the current viewBox (defaults to `APAC_VIEW`, with a `WORLD_VIEW` preset). **Dots sit at their true `lon`/`lat`** (`buildDots`) — co-located deals are grouped into a tiny on-location rosette (≤~0.4°, capped so they stay on the city), and the card shows **prev/next nav** to step through all deals at one location, so nothing is scattered onto open water. Dot radius is recomputed per zoom (`applyView`) to stay ~constant pixel size; wheel-zoom is gated behind Ctrl/⌘ so page scroll passes through, and zoom-in reaches ~city level (`mapView.w` floor of 8 / 1000) so tight stacks resolve into individual bubbles. Arcs are plain dashed lines — no SVG arrowhead markers (they scale with the viewBox and balloon when zoomed in). Click a dot → `selectDot()` opens the `#dealcard` detail panel (with a "filter table to country" action). `initMapControls()` wires pan/zoom/buttons once.
- Shared (not per-week): `WIDER`, `GLOSS`, and `SECONDARY` (the May 2026 secondary-loan composite). Each `SECONDARY` row is `{name, region, bid (price /100), chg (MoM bid change)}` plus optional `link:{wk, borrower}` when the name also appears in a weekly reading. The panel renders: the per-name bid bars (`fillSecondary`), a zone-coloured **bid-distribution histogram** (`fillSecDist`), a diverging **biggest-movers** chart from `chg` (`fillSecMovers`), and a **primary↔secondary cross-link** block (`fillSecLinks`) that pairs each `link`ed name's primary deal (looked up in `WEEKS[wk]`) with where its paper trades. All four are rendered once at init (secondary is week-independent).
- **UX / accessibility layer:** header **Present** button toggles `html[data-present="on"]` (bigger type + higher contrast for screen-sharing); a `@media print` block + `.print-omit` tags export a clean briefing one-pager (drops chrome, deal map, deal table, secondary, glossary). KPI cards take an optional `spark:"vol|deals|ctbc|ai|brokerage|spread"` (or an explicit `[a,b,c]`) → a 3-week sparkline from real cross-week series (`weekSeries`/`sparkSVG`). Deal-table rows are expandable (`tr.drow` → hidden `tr.ddetail`), keyboard-operable (Tab/↑↓ between rows, Enter/Space to expand; sortable headers are `role=columnheader` with `aria-sort` + Enter), and the filtered+sorted set exports via the **CSV** button (`exportCSV` → `filteredDeals`, Blob download, no CDN). Sector pills get inline-SVG icons (`SECTOR_ICON`/`secIcon`); week-switch plays a `.weekswap` fade (respects reduced-motion).
- **To add a week:** copy a sibling `src/data/<week>.json`, set its `key`/`tab`/`date` and fill the `week` object, then `npm run build && npm test`. The build picks the new file up automatically and re-sorts `ORDER` by `date` — no other file changes, and `current` follows to the newest. Diff the new week's `whatsnew` against the prior week. **Every deal needs `theme`, `spreadBp`, `basis`** (or it drops from the Trends charts) and **`lon`, `lat`, `loc`, `locType`** (or it drops from the map — geocode the borrower's HQ, or the project site for project finance, to city-level decimal degrees). Tag any CTBC cross-border deal with `flowFrom:"Taiwan"` (or its origin country) so its arc draws.
- **After editing, validate:** `npm run verify` (build + jsdom smoke test). Without Node: `python3 tools/build.py`, then open `dist/CTBC_Loan_Market_Dashboard.html` in a browser and confirm no console errors / no network requests, the date toggle swaps the table, Trends charts render, and the map draws dots + CTBC arcs (click a dot → detail card).
